What is React

A JavaScript library for building user interfaces.

It was built in Facebook Inc by Jordan Walke.

It got deployed on Facebook in 2011 and on Instragram in 2012.

It got open sourced on JSConf US 2013.

OK, what the fuck is React?

  • Virtual DOM
  • JSX
  • One-way data flow

Where can I use React?

  1. In your front-end JavaScript applications for your components
  2. In your back-end JavaScript (Node.js) code to pre-render your components

When should I use React?

React is a "heavy" library.

It sports rich syntax and sizes 150KB minified and gzipped.

You should use it in front-end applications with:

  1. Lots of different components
  2. Intense data flows

Components

In React components are prototypes of DOM elements.

Components relate to elements, similar to how classes relate to objects.

A simple example

Let's create a "greeting" component, that will greet us using our name.

var Greeting = React.createClass({
    render: function() {
        var tag = 'div';
        var properties = {
            // We use "className" because "class" is a reserved keyword
            className: 'greeting'
        };
        var content = 'Hello ' + this.props.name;

        return React.createElement(tag, properties, content);
    }
});

var mountNode = document.getElementById('greeting-container');

React.createElement(Greeting, { name: 'Jane' });

JavaScript upgrade to ES3987

The React team adopts the latest JavaScript syntax (ES6/ES2015/ES2017).

It adopts new features like:

  • JavaScript classes (e.g. class A extends B)
  • Template literals
  • Arrow functions (e.g. (a, b) => (a + b))

Then use tools to transpile this into ES5, which is considered compatible with all browsers.

A simpler example

Simplifying our example by using JavaScript classes and template literals.

class SimplerGreeting extends React.Component {
    render() {
        var tag = 'div';
        var properties = {
            // We use "className" because "class" is a reserved keyword
            className: 'greeting'
        };
        var content = `Hello ${this.props.name}`  // Template literals

        return React.createElement(tag, properties, content);
    }
}

var mountNode = document.getElementById('simpler-greeting-container');

React.createElement(SimplerGreeting, { name: 'Jane' });

JSX

JSX is a DSL (Domain Specific Language) for React.

It makes HTML code valid JavaScript expressions.

So now HTML code is the same to JavaScript as numbers, objects, function etc.

Let's see what that means.

An even simpler example

JSX helps us replace the confusing React.createElement method (and more).

class EvenSimplerGreeting extends React.Component {
    render() {
        return <h1>Hello, {this.props.name}</h1>;
    }
}

var mountNode = document.getElementById('simpler-greeting-container');

ReactDOM.render(<EvenSimplerGreeting name="Even Simpler Jane" />, mountNode);
